📝 Коли потрібно створювати звернення

🐛 Звернення типу BUG

Створюйте Bug, якщо:

  • ✗ функціонал працює некоректно або система не зберігає дані
  • ✗ з'являється повідомлення про помилку
  • ✗ кнопка, поле або форма не працюють як очікується
  • ✗ поведінка системи відрізняється від очікуваної
  • ✗ проблема блокує або ускладнює робочий процес

✨ Звернення типу IMPROVEMENT / ENHANCEMENT

Створюйте Improvement, якщо:

  • 🌟 потрібно покращити поточний функціонал
  • 🌟 є пропозиція щодо зміни логіки роботи
  • 🌟 потрібно зробити процес зручнішим або швидшим
  • 🌟 потрібно додати нову поведінку або спростити існуючу

📎 Що додавати до заявки

  • 🔗 Посилання на сторінку або модуль
  • 📸 Скріншот з обведенням проблемної області
  • 🎥 Відео або Jam для складних сценаріїв
  • 📋 Опис фактичної та очікуваної поведінки
  • ⚠️ Інформація, чи блокує проблема роботу

✅ Стандарт якісної заявки

🎯 Якісна заявка повинна містити:

✓ Чекліст для якісної заявки:

  1. 📌 Summary — чіткий, зрозумілий, конкретний
  2. 📝 Description — короткий опис проблеми
  3. ❌ Фактична поведінка — що відбувається зараз
  4. ✅ Очікувана поведінка — що мало б відбуватися
  5. ⚠️ Пріоритет — чи блокує робочий процес
  6. 🔗 Посилання — на сторінку / модуль
  7. 📎 Вкладення — скріншот / Jam / видео
❌ Помилки, яких потрібно уникати:
  • Занадто загальний Summary ("не працює", "щось зламалось")
  • Фрази без контексту та деталей
  • Відсутність посилання на сторінку
  • Без скріншота / Jam / відео
  • Не описана фактична / очікувана поведінка

⚠️ Типові помилки при створенні заявок

❌ Помилки в Summary:

  • "Не працює"
  • "Щось зламалось"
  • "Вибиває помилку"
  • "Потрібно змінити"

✅ Правильний Summary:

  • "Не зберігаються дані в таблиці"
  • "Помилка 500 при натисканні кнопки"
  • "Фільтр за датою не працює"
  • "Додати редагування форми"

❌ Помилки в Description:

  • Тільки слово "баг"
  • Занадто короткий опис
  • Без деталей
  • Не вказано контекст

✅ Правильний Description:

  • Коротко, але ясно
  • З деталями і кроками
  • З контекстом проблеми
  • З посиланням на сторінку

🐛 Як створити Bug

Коли це використовується

⚠️ Створюйте Bug, якщо:

  • ✗ функціонал не працює
  • ✗ система видає помилку
  • ✗ дані не зберігаються
  • ✗ результат не відповідає очікуваному
  • ✗ проблема блокує вашу роботу
1️⃣

Оберіть тип Bug на Home Help Center

Натисніть на блок Bug, щоб відкрити форму.

2️⃣

Заповніть поле Summary

Коротко та конкретно вкажіть суть проблеми.

✅ Хороший Summary:
  • Не зберігаються зміни в таблиці
  • Після натискання кнопки помилка 500
  • Не працює фільтр за датою
❌ Поганий Summary:
  • Не працює
  • Помилка
  • Баг у формі
3️⃣

Заповніть поле Description

Коротко поясніть, у чому проблема.

📌 Приклад:
Під час редагування таблиці вантажів після внесення даних і натискання кнопки застосування система не зберігає зміни та повертає помилку.
4️⃣

Вкажіть фактичну поведінку

Опишіть, що саме відбувається зараз.

📌 Приклад:
Дані не вносяться у форму. Після натискання "Застосувати" система не зберігає значення та показує помилку.
5️⃣

Вкажіть очікувану поведінку

Опишіть, що мало б відбуватися.

📌 Приклад:
Після натискання "Застосувати" внесені дані повинні коректно зберігатися у формі без помилки.
6️⃣

Вкажіть пріоритет: чи блокує робочий процес?

Оберіть:

  • 🔴 Yes — якщо без цього неможливо продовжити роботу
  • 🟡 No — якщо є обхідний шлях або менші труднощі
7️⃣

Додайте скріншот або Jam-посилання

Для складних сценаріїв або динамічних помилок:

  • 📸 Додайте скріншот через Lightshot
  • 🎥 Або запишіть Jam-посилання
💡 Коли Jam особливо корисний:
  • Проблема не відтворюється одним скріншотом
  • Важливо показати послідовність дій
  • Баг виникає не одразу
8️⃣

Додайте посилання на сторінку

🔗 Вставте пряме посилання на сторінку / модуль, де виникла проблема. Це допоможе технічній команді швидше перейти до потрібного місця.

9️⃣

Перевірте форму ще раз

✓ Переконайтесь, що всі поля заповнені та все зрозуміло. Це останній момент для правок!

📸 LIGHTSHOT — Швидкі скріншоти з позначками

Для чого використовувати Lightshot

⚡ Lightshot зручно використовувати, якщо:

  • ✓ Достатньо одного скріншота
  • ✓ Проблема візуально помітна на одному екрані
  • ✓ Потрібно швидко обвести конкретне поле або кнопку
  • ✓ Потрібно додати стрілки, рамки або короткий коментар
⚡ Lightshot дозволяє:
  • 🎬 Зробити скріншот кількома кліками
  • ✏️ Обвести проблемну область
  • 🖍️ Додати стрілки і текст
  • 📋 Одразу вставити в Jira (Ctrl+V / Cmd+V)

Встановлення Lightshot

1️⃣

Перейдіть на сторінку Lightshot

2️⃣

Встановіть розширення

⬇️ Натисніть "Add to Chrome" (або для вашого браузера).

3️⃣

Закріпіть розширення у панелі

📌 Натисніть на іконку розширень у верхній панелі → знайдіть Lightshot → натисніть pin.

4️⃣

Перевірте іконку у панелі

✅ Іконка Lightshot тепер завжди видима у верхній панелі браузера. Готово!

Як зробити скріншот через Lightshot

1️⃣

Натисніть на іконку Lightshot

🎬 Іконка у верхній панелі браузера. Натисніть на неї.

2️⃣

Виділіть потрібну область

🖱️ Клікніть і перетягніть, щоб обрати ділянку екрана для скріншота.

3️⃣

Додайте позначки (за потреби)

✏️ Додайте:

  • ➡️ Стрілки
  • ⬜ Рамку або обведення
  • 📝 Текст з поясненням
  • 🟨 Підсвітку
4️⃣

Збережіть або скопіюйте

💾 Натисніть "Copy" для копіювання, потім вставте в Jira (Ctrl+V / Cmd+V). Просто і швидко!

💡 Рекомендація для Bug-репортів:

🖍️ Обводьте проблемне поле, кнопку, повідомлення про помилку або неправильну ділянку інтерфейсу. Це помітно допоможе технічній команді зрозуміти, де саме виникає проблема.

🎥 JAM.DEV — Запис багів з повним контекстом

Для чого використовувати Jam

✅ Jam рекомендується, коли:

  • 🎯 Проблему складно пояснити словами
  • 🔄 Є кілька кроків до помилки
  • 📹 Потрібно показати сценарій від початку до збою
  • 🖱️ Проблема пов'язана з формами, кліками, переходами, нестабільною поведінкою
✅ Jam дає технічній команді максимум контексту

Замість описання словами можна просто показати повний сценарій — команда все одразу зрозуміє и почне аналіз набагато швидше!

Встановлення Jam

1️⃣

Перейдіть на сторінку Jam

2️⃣

Встановіть розширення

⬇️ Натисніть "Add to Chrome" (або для вашого браузера).

3️⃣

Перевірте іконку у панелі браузера

📍 Після встановлення іконка Jam повинна з'явитися у верхній панелі браузера.

Як записати проблему в Jam

1️⃣

Натисніть на іконку Jam

🎬 Іконка знаходиться у верхній панелі браузера. Натисніть на неї.

2️⃣

Авторизуйтесь (якщо потрібно)

🔐 Якщо ви вперше, натисніть Login та авторизуйтесь через Google, email або інший спосіб.

3️⃣

Оберіть "Record Tab"

⏹️ Натисніть на кнопку Record Tab.

4️⃣

Виберіть область для запису

📺 Виберіть Tab (поточна вкладка) або Desktop (весь екран).

5️⃣

Дайте дозвіл на запис екрана

✅ Дозвольте браузеру записувати екран. Дозвіл потрібен тільки один раз.

6️⃣

Запишіть проблему крок за кроком

⏱️ Перед вами з'явиться панель запису з таймером. Повторіть проблему від початку до кінця.

7️⃣

Завершіть запис

⏹️ Натисніть кнопку зупинки запису.

8️⃣

Отримайте посилання Jam

🔗 Натисніть Create & copy link, щоб скопіювати посилання на запис.

9️⃣

Вставте посилання в Jira

📋 Перейдіть до заявки в Jira Service Desk і вставте Jam-посилання у поле Links або Description.

🤔 Що обрати: Jam чи Lightshot?

⚙️ Параметр 📸 Lightshot 🎥 Jam
⏱️ Час на створення ⚡ 30 секунд ⏳ 2–5 хвилин
📌 Коли використовувати Одна картинка, чіткий фокус Динаміка, послідовність дій, контекст
🎯 Складність проблеми Проста, видна на 1 екрані Складна, багато кроків
✏️ Можливість позначок ✅ Так (стрілки, рамки, текст) ❌ Ні (але запис говорить сам за себе)
🔊 Можливість звуку ❌ Ні ✅ Так (можна записати коментар)
🏆 Найкращий варіант для критичних дефектів:

💎 Скріншот (Lightshot) + Jam-посилання + Пряме посилання на сторінку

Такий комбо значно підвищує якість заявки і дозволяє технічній команді одразу розпочати аналіз без зайвих питань!

❓ FAQ — Часто задавані питання

❓ Що робити, якщо не приходить лист для встановлення пароля?
📧 Перевірте папки Spam, Promotions, Інше. Якщо листа немає, спробуйте повторно використати форму входу або звертайтесь до відповідального за інтеграцію. Зазвичай процес займає 5-15 хвилин.
❓ Що робити, якщо проблема складна і важко описати?
🎥 Використайте Jam — запишіть весь сценарій від початку до проблеми. Це дасть технічній команді повний контекст, і вони швидко зрозуміють проблему без додаткових питань.
❓ Що робити, якщо достатньо одного зображення?
📸 Використайте Lightshot. Зробіть скріншот, обведіть проблемну область та вставте у форму заявки (Ctrl+V / Cmd+V). Це займе максимум 30 секунд!
❓ Що робити, якщо проблема блокує роботу?
🔴 У полі "Чи блокує робочий процес" обберіть Yes та додайте максимум контексту: посилання, скріншот, Jam або відео. Також повідомте відповідального менеджера для пришвидшення.
❓ Можна додавати кілька Jam-посилань в одну заявку?
✅ Так, можна! Якщо проблема проявляється в кількох місцях, додайте кілька записів. Але рекомендується не більше 3–5 для читабельності та швидкості аналізу.
❓ Як довго очікувати відповіді на заявку?
⏱️ Залежить від пріоритету:
  • 🔴 Блокуючий дефект: в той же день
  • 🟡 Звичайний дефект: 1–2 дні
  • 🟢 Improvement: за графіком спринту
❓ Як відстежити статус заявки?
📊 Перейдіть в розділ "Запити" у меню профілю. Там буде список усіх ваших заявок зі статусами: Open, In Progress, Resolved, Closed. Ви завжди можете бачити актуальний статус та читати коментарі команди.
